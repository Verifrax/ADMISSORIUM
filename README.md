# ADMISSORIUM

ADMISSORIUM is the admissibility enforcement implementation surface for VERIFRAX.

It materializes org-wide checks, repair proposals, quarantine records, and merge-blocking behavior that keep public repositories, hosts, packages, and projections subordinate to the accepted VERIFRAX constitutional object graph.

ADMISSORIUM does not decide truth.

ADMISSORIUM refuses inadmissible materialization.

## Status

```text
SYS-ADMISSORIUM

SOVEREIGN_CHAMBER: false

````

## Role

ADMISSORIUM exists because VERIFRAX has crossed the point where repo-by-repo discipline is enough.

The public perimeter contains many materialized surfaces:

* repositories
* hosts
* packages
* READMEs
* license files
* evidence objects
* authority objects
* execution receipts
* verification results
* recognition objects
* recourse objects
* generated host pages
* status surfaces
* docs surfaces
* package pages
* public cache and crawler-visible surfaces

ADMISSORIUM enforces admissibility across those surfaces.

It is not a bot.

It is not a sovereign chamber.

It is the operational actuator for the admissibility gate.

## Operating doctrine

```text
A thing is not VERIFRAX-real because it exists.
A thing is not VERIFRAX-real because it merged.
A thing is not VERIFRAX-real because it was published.
A thing is VERIFRAX-real only if the accepted object graph admits it.
```
